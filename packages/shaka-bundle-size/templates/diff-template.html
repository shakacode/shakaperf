bundle-size-diffs/

What is this file?
When you create a Loadable Component, webpack will create one or more JS chunks for it.
These JS chunks can be shared between different components.

This file shows:
  • How many chunks are created for each Loadable Component (fewer chunks are better)
  • Their cumulative size:
    • brotliSizeKb - size when compressed by Brotli. Brotli is more efficient than gzip, but the resulting size may randomly fluctuate, making it a poor choice for A/B tests as it can trigger false regressions. Brotli compression is used in production.
    • gzipSizeKb - size when compressed by gzip. Gzip is less efficient, but the resulting size is more stable. Gzip compression is used in the bundle-size check only.
This file is a source map showing the list of source files composing each JS chunk for every Loadable Component.

By looking at the diff below, you can:
  • See which source files are responsible for the growth of a specific Loadable Component
  • Identify which JS chunks are shared between different Loadable Components
How to compare locally

Tired of waiting for slow CI? Run the comparison locally:

  1. Build your project first (command may differ, but should look like this): 
    bin/shakapacker --json public/webpack/production/webpack-stats.json
  2. Download baseline from main branch: 
    yarn shaka-bundle-size --config path/to/bundle-size.config.ts --download-main-branch-stats
  3. Compare current build against baseline: 
    yarn shaka-bundle-size --config path/to/bundle-size.config.ts --compare

Use your package manager (yarn, npm, pnpm, etc.) to run the commands.
